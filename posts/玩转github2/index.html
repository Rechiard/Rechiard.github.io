Git的安装以及基本使用

  1. 下载Git

    目的：通过git管理github托管项目代码

    Git官网下载：https://www.git.scm.com/download/win

    双击安装

  2. Git工作区域

    1. Git Repostitory（Git 仓库）

    最终确定的文件保存到仓库，成为一个新的版本，并且对他人可见

    1. 暂存区

    暂存已经修改的文件最后统一提交到git仓库中

    1. 工作区（Working Directory）

    添加、编辑、修改文件等动作

  3. 向仓库添加文件流程

  4. 初始化git(本地操作)————（工作区->暂存区）

    1. 基本信息设置

      1. 设置用户名

        git config --global user.name 'XXXXX'

      2. 设置用户名邮箱

        git config --global user.email '616316004@qq.com'

    2. 初始化一个新的git仓库

      1. 创建文件夹

        mkdir 文件名

      2. 在文件内初始化git（创建git仓库）

        进入目标文件夹：cd 文件名

        初始化本地git仓库（暂存区）：git init

    3. 向仓库添加文件

      1. 添加到本地仓库（暂存区）：

        git add 文件名.后缀

      2. 查看：

        git status

      3. 将文件从暂存区提交到github仓库

        git commit -m '提交状态信息说明'

    4. 修改仓库文件

      1. 将文件的内容修改

      2. 重新进行第三个步骤

    5. 删除仓库文件

      1. 删除文件

        rm 文件名.后缀

      2. 从Git中删除文件

        git rm 文件名.后缀

      3. 提交操作

        git commit -m '提交描述'

  5. Git管理远程仓库————（暂存区->Git Repository）

    使用远程仓库的目的：备份，实现代码共享集中化管理

    1. 添加远程地址 命令：git remote add origin https://github.com/Rechiard/ProjectCollection.git （远程地址）

    2. 添加到本地仓库（暂存区）：

      git add 文件名.后缀

    3. 将文件从暂存区提交到github仓库

      git commit -m '提交状态信息说明'

    4. 将本地仓库同步到git远程仓库

      git push origin master

    5. 报错：The requested URL returned error:403 Forbidden while accessing或没有权限

      回答：是因为这是私有项目，没有权限，输入用户名密码，或者远程地址采用这种类型：

      vi .git/config
      
# 将
[remote "origin"]
  url = https://github.com/用户名/仓库名.git
修改为：
[remote "origin"]
  url = https://用户名:密码@github.com/用户名/仓库名.git

  6. Git克隆操作

    目的：将远程仓库（github对应的项目）

    1. 通过github仓库的Clone or download复制链接

    1. 克隆：git clone 复制的链接

      查看：git config --list
