如何快速搭建个人博客网站
@ Author Gongjianbo | Wednesday, Nov 25, 2020 | 1 minute read | Update at Wednesday, Nov 25, 2020

本项目为记录我的第一次利用hugo框架建立自己的个人博客过程

通过B站的CodeSheep（程序羊）的视频教学认识到了博客的个人搭建过程，整个搭建过程大概可以分为 个步骤：

  • 1.下载并部署git(一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理)

    • 1.1 git官方下载链接
    • 1.2 git基本命令教学网站

  • 2.下载Hugo并配置环境变量path

    • 2.1 从Github上下载hugo.exe压缩包（window方法）
    • 2.2 通过brew工具(MacOS上的包管理工具，可以简化 macOS 和 Linux 操作系统上软件的安装)下载hugo(通过指令：brew install hugo)
    • 2.3 需要在新建一个文件夹名为hugo，里面还有再新建一个bin的文件夹再放入下好的hugo.exe版本，在系统变量path中添加到…/bin路径

  • 3.建立个人博客

    • 3.1 hugo建立网站命令：hugo new site [网站名称]
    • 3.2 添加网站主题命令：git clone [主题网址]

  • 4.在本地启动个人博客

    • 4.1 hugo在本地启动个人博客命令：hugo server -t [主题名称] --buildDrafts
      • 注：启动个人博客时需要在相应的博客文件夹内执行此命令
    • 4.2 通过cmd得到的网址检验网址是否启动成功

  • 5.新建一篇博客文章

    • 5.1 通过hugo命令：hugo new post/[markdown文件名称].md
    • 5.2 将需要展示的内容编辑到新建的文件中，致此来完成需要发布的博客文章

  • 6.将个人博客部署到远端服务器(Github免费服务器)

    • 6.1 创建新仓库，名称必须是[owner的名称].github.io
    • 6.2 通过hugo命令：hugo --theme=[主题名称] --baseUrl="[https://owner的名称.github.io/]" --buildDrafts
    • 6.3 在项目文件夹内会生成一个public文件,检验成功后，进入public文件夹中，
      • 通过命令：
      • 1.git init
      • 2.git add .
      • 3.git commit -m "我的hugo博客第一次提交"
      • 将public上传到github的仓库中
    • 6.4 进行远地和本地的关联，通过命令：git remote add origin https://github.com/[owner的名称]/[onwer的名称].github.io.git
    • 6.5 推送到远端的master上，通过命令：git push -u origin master，再输入github的账号密码
    • 6.6 去刚新建的仓库中检验一下是否提交成功

  • 7.通过远端生成的网址访问个人博客

    • 7.1 博客网站为：http://[owner的名称].github.io
